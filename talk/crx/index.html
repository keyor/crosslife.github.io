Manifest文件

  • 必须有

  • JSON格式(manifest.json)

  • 描述了关于扩展的各种重要信息

  • 格式文档

{
  "manifest_version": 2,
  "name": "My Extension",
  "version": "1.0.0",
  "description": "The best extension",
  "icons": {...},
  ...
}

Broswer Actions

  • badge
  • popup
// in manifest.json
{
    'browser_action': {
        'default_icon': 'icon.png',
        'default_popup': 'popup.html'
    },
    ...
}

Content Scripts

  • 一些在Web页面里运行的JS代码，通过共享的DOM与页面交互(♆)

  • 隔离的环境，不能访问页面里定义的全局变量

// in manifest.json
{
  "content_scripts": [
      {
        "matches": ["http://www.douban.com/*"],
        "css": ["mystyles.css"],
        "js": ["jquery.js", "myscript.js"]
      }
  ],
  ...
}

Background Pages & Event Pages

  • 在后台运行的脚本

  • 通常用来监听一些事件

Background Pages

  • 从浏览器启动就开始在后台运行
// in manifest.json
{
  "background": {
      "scripts": ["background.js"]
  },
}
  • 尽量用Event Pages取代

Event Pages

  • 按需加载(只有需要的事件触发时，才会被执行)
// in manifest.json
{
  "background": {
      "scripts": ["eventPage.js"]
      "persistent": false
  },
}
